## Design Context

### Users
个人博客博主，主要发布：
- **技术文章**：代码教程、技术笔记、开发经验分享
- **作品展示**：项目展示、作品集、设计案例

用户场景：在 GitHub Pages 上部署个人博客，面向技术读者和潜在合作者。

### Brand Personality
**简约优雅** — 三个词概括：
- **专注**：去除干扰，让读者沉浸于内容
- **专业**：技术可信，作品可信
- **克制**：不多余、不张扬、恰到好处

情感目标：让访客感到"这是一位认真做事的人，值得信赖"。

### Aesthetic Direction

**视觉基调**：
- 简约但不冷清，优雅但不浮夸
- 以内容为中心，UI 不抢焦点
- 技术文档的严谨 + 作品展示的精致

**主题模式**：
- 支持手动切换 Light / Dark 模式（保留现有切换功能）
- 浅色模式：清爽明亮，适合日间阅读
- 深色模式：沉稳护眼，适合长时间技术阅读

**配色参考**（基于现有 variables.scss）：
| 模式 | 背景 | 强调色 | 文字 |
|------|------|--------|------|
| Light | `#f5f5fa` | `#34495e` | `#707070` |
| Dark | `#303030` | `#ecf0f1` | `rgba(255,255,255,0.7)` |

**字体优先级**：中文优先
- 正文：思源宋体 / 思源黑体作为首选（需配置 CDN）
- 代码块：保留 Menlo/Monaco/Consolas

### Design Principles

1. **内容至上**
   - 卡片设计承载内容，不喧宾夺主
   - 间距足够但不浪费空间
   - 字号适合中文阅读（1.6rem+）

2. **阅读舒适**
   - 行高 1.85 保持中文阅读流畅
   - 深色模式降低蓝光，适合长时间阅读技术文章
   - 代码块与正文有明确视觉区分

3. **作品可展示**
   - 支持图片、代码、Mermaid 图表
   - Tile 样式适合作品集首页展示
   - 响应式适配手机/平板查看

4. **交互克制**
   - 过度动画降至最低（仅保留必要的 hover 反馈）
   - 阴影层次微妙（shadow-l1/l2）
   - 不追求"炫酷"，追求"好用"

5. **中文优化**
   - 利用现有 `text-autospace` CJK 间距优化
   - 考虑引入思源字体提升中文排版
   - 标签、分类等 UI 元素适配中文长度

### Existing Design Assets

**设计令牌（已定义于 `variables.scss`）**：
- Spacing: `xs(5px) → 3xl(40px)`
- Card: `border-radius: 10px`, `padding: 20-30px`
- Shadows: 多层柔和阴影（l1/l2）
- Breakpoints: `sm→md→lg→xl→2xl` (640→1536px)

**布局模式**：
- 左侧边栏：头像 + 站点信息
- 主内容区：文章卡片列表
- 右侧边栏：Widget（标签云、归档、TOC）

**功能模块**：
- 文章三种样式：default / compact / tile
- Mermaid 图表支持
- Alert 提示块（note/tip/warning 等）
- 代码块复制按钮
- 图片响应式处理

### Recommendations for Personalization

1. **字体定制**
   - 在 `baseof.html` 添加思源字体 CDN 引用
   - 或在 `variables.scss` 调整 `--base-font-family`

2. **配色微调**
   - 强调色可改为更符合个人偏好的色系
   - 如技术博客可选蓝绿系，作品展示可选暖色系

3. **侧边栏头像**
   - 替换为个人头像/Logo
   - 可配置站点副标题（中文描述）

4. **Widget 配置**
   - 归档 Widget 适合技术博客时间线
   - 标签云适合分类技术主题

### Anti-References
**避免的方向**：
- 杂志式密集排版（不适合技术长文）
- 过多装饰性动画
- 高饱和度色彩冲突
- 小字号挤压排版（中文阅读不友好）
